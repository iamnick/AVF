Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

There are already lots of sites, services, and games out there that give an option to login through Facebook, and doing so with Twitter would be similar. Logging in through your social media account would make it easier for the user, since there wouldn’t be a new account/password to keep track of. Users could also easily share things from the site/service/game easily as well via their social media account since the two would be connected already. Similarly, you could make payments by logging into your Twitter account too and using your information stored there.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

We need to use JSON-P to get around the same origin policy, since our apps would be requesting data from a different domain than the server it’s located on.

Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

I think many of the social media APIs would be great with the camera, as sharing photos and video is something most people like to do. These APIs could also pair well with the geo location.

Please describe any design and functionality intentions you may have for your Demo app components this month.

For my demo app this month, I’d like to demo as many native features as I can in some way just to learn how to access and use them all. In particular, I’m interested in using the accelerometer as I think tilting and moving the device to do things is pretty cool.
